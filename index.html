Project Type

Undergraduate thesis, Ionian University

Design Challenge

The main goal was to create a local multiplayer game with almost zero setup friction. Instead of requiring dedicated controllers, installations, or complex pairing, I designed the system so players could join instantly by scanning a QR code and using their phones as controllers.

Key Contributions

Real-time controller system

Built an asynchronous communication pipeline with PHP and AJAX to connect multiple mobile devices to the projected game screen.

Gameplay and systems design

Designed the combat loop, playable classes, and the bird-based power-up and penalty delivery system.

Projection calibration tool

Created an admin dashboard that aligned spawn points and game elements with real architectural features such as windows.

Process & Decisions

Why web instead of Unity

I chose a web-based stack to maximize accessibility and cross-device compatibility. This let players join instantly through the browser without plugins, downloads, or platform-specific setup.

Designing for buildings

To make characters feel physically embedded in the projection surface, I implemented a slider-based calibration system that matched digital spawn points to real windows and facade features.

Performance optimization

Testing on lower-power hardware revealed that continuous AJAX polling was too expensive. I refactored the refresh loop from frame-rate-tied updates to a throttled 100ms interval, reducing CPU load while keeping gameplay responsive.

Platform constraint

During development, I had to troubleshoot iOS browser restrictions around gyroscope access, which required specific HTTPS permissions before motion controls would function reliably.

Outcome

The project was showcased at the 16th Audiovisual Arts Festival in 2023 and tested in a public Game Hub setting. The strongest result was how naturally people understood the QR-code-to-controller flow and how intuitive the gyroscope aiming felt, especially for non-gamers.

Public user testing supported the overall playability and usability of the experience, but for portfolio purposes the most important takeaway is that the concept worked successfully in a real exhibition environment.

Reflection

This project taught me how to design a game across both digital and physical space, balancing systems design, technical implementation, and public installation constraints. If I continue it, I would automate QR generation from the server IP and add active disconnect handling so player slots free up immediately when a browser closes.

Project 02

The Unfortunates

A narrative-heavy “cozy dystopia” adventure set in a surreal branch of purgatory, where players guide newly arrived spirits toward reincarnation through conversations and small but emotionally charged choices.

Developer Game Design Game Systems Animations Balancing

Built as a semester-long team project in Unity, the game explores a design contradiction: how to make player choices feel meaningful inside a story that ultimately argues fate is unavoidable. My work focused on gameplay programming, dialogue systems, iteration, and making narrative interactions feel more responsive and alive.

Design Challenge

The core challenge was balancing player agency with a deliberately nihilistic narrative. We wanted conversations and judgments to feel emotionally significant even when the larger story suggests that none of the player’s decisions can truly escape fate.

My Contributions

Developer

Implemented gameplay systems in Unity, including 2D platforming movement, parallax effects, environmental interactions, and event-based dialogue actions that connected narrative beats to in-game behavior.

Game Design

Helped shape a “cozy dystopia” experience where relaxed traversal and melancholic storytelling reinforce each other, with choices designed to provoke reflection rather than branch into radically different outcomes.

Game Systems

Co-developed the dialogue system and the end-game statistics system, giving the player feedback on their choices and strengthening the sense that their actions had emotional and thematic weight.

Animations

Worked on polish and motion feedback, including changes that made environmental props and interactions feel intentional and atmospheric rather than static or bug-like to players.

Balancing

Used playtesting feedback to adjust pacing, interaction variety, and the rhythm of dialogue events so the experience stayed emotionally engaging instead of repetitive or overly predictable.

Gallery

Process & Iteration

Concept development

The team used a modified brainwriting approach to quickly generate ideas around the contradictory theme “Cozy Dystopia,” which helped establish the project pillars of cozy gameplay and a dystopian story.

Narrative shift

The original bureaucratic office setting evolved into an ethereal floating-island purgatory, which better supported both the surreal tone and more expressive level design.

Dialogue tooling

To support branching conversations efficiently, I worked with a custom GraphView-based dialogue tool that allowed the team to author text while also triggering gameplay-specific events such as hugs and other scripted moments.

Breaking predictability

After feedback suggested that the dialogue loop became too formulaic, we introduced “break-the-formula” moments and special event commands to make interactions feel more intimate, surprising, and memorable.

Player readability

Early testers misread some visual elements, such as judgment hooks, as bugs. Instead of removing them, we added motion and polish so they felt atmospheric and intentional inside the world.

Game feel and polish

We refined simple mechanics with audiovisual feedback, including reactions for landing and environmental ambience, to make the space feel responsive despite the small scope of the interaction model.

Outcome

The Unfortunates succeeded as a polished small-scope narrative game and received a strong immersion rating in the final evaluation. Players described the experience as cozy, dark, mysterious, philosophical, and emotionally engaging, which aligned closely with the intended tone.

One of the most useful design takeaways was that the statistics screen helped give retrospective meaning to player choices, especially in a game where the narrative intentionally limits direct agency.

Reflection

This project taught me how much “polish” shapes player perception, especially in a dialogue-driven game with relatively simple mechanics. I also learned how useful custom tools can be when they help narrative design, implementation, and iteration happen in the same workflow.

If I continued the project, I would expand the reincarnation outcomes with more visible transformation feedback so players could see the consequence of each spirit’s final fate more directly.

Project 03

Loose Ends

A time-travelling murder mystery set in a steampunk-inspired 18th-century France, where two thieves use disguises, stealth, and social engineering to untangle a noble conspiracy across repeated loops.

I led the design of the disguise and dialogue systems, shaped the palace level layout for a 3D world with a 2D side-scrolling perspective, and helped turn the mystery into a knowledge-based progression structure instead of a combat-driven one.

Project Type

Course project for “Interactive and linear storytelling with digital media”

Design Challenge

The main challenge was balancing player freedom inside a time loop with a mystery that still felt coherent and readable. We wanted progression to come from information, timing, and social manipulation rather than combat skill or reflex-heavy play.

My Contributions

Disguise System

I designed and implemented the core identity-switching mechanic, where taking an NPC’s role changed dialogue options and access to restricted spaces.

Dialogue Logic

I built a keyword-based dialogue structure where players unlocked topics through eavesdropping, letters, and discoveries across loops.

Level Design

I designed the palace layout around a 3D world with a 2D side-scrolling view, using class-divided routes like service passages, vents, and hidden lower-status areas.

Process and Iteration

Disguise as key-and-lock logic

I structured disguises as the main progression layer, so identities acted like social keys that opened conversations, rooms, and clues.

Conspiracy board

I helped shape a persistent evidence board that linked documents, overheard facts, and discovered relationships to reduce confusion between loops.

Quest redesign

I pushed an assassination-focused revenge path into sabotage and humiliation, which made the story more thematic and supported stealth-driven play.

Environment storytelling

I designed spaces like the Guest Room and Jeweler’s Room to support the Ring Heist quest with multiple approaches, stealth routes, and class-based contrast.

Gallery

Outcome

The project proved that a complex narrative web could still feel playable when information was treated as progression. The strongest result was the “aha” moment when players reused knowledge from one loop to bypass or shortcut challenges in the next.

Reflection

Playtesting showed that some narrative beats were too easy to physically reach before players understood their meaning, which created confusion. That taught me to gate critical path understanding through knowledge, not only through space, sequence, or item pickup.

Next Improvement

I would improve failure feedback with a clearer “time-up” UI and more contextual guidance from the partner character, so players better understand why a loop failed and where to investigate next.
